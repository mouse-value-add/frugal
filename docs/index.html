The insight

Most AI tasks are over-routed. You're paying for capability you don't use on 60–80% of your calls — and when you do need capability, a bare chat model is rarely the answer. The expensive part isn't the model. It's the wrong path: a frontier API when a local model plus search would have worked, long-context when retrieval would have, an agent loop when one tool call would have. Frugal classifies each request and routes to the cheapest reliable toolchain that clears the quality bar.

Toolchain classifier
Detects the task and the capabilities required — local model, hosted model, search, browser, code execution, extraction, embeddings, semantic cache, or a multi-step combination.
Cheapest-path router
Per task and quality tier, picks the cheapest reliable toolchain that clears every threshold. One runtime, one routing decision, shared between CLI and MCP server.
Eval as source of truth
Use-case benchmarks decide which toolchains win. Routing isn't opinion — it's what the data says wins for your workload, rerun on every release.

How it works

frugal run "summarize this repo"
       │
       ├─ classifies task → recipe ID + quality tier
       ├─ loads recipe step list from config/use_cases/<id>.yaml
       ├─ for each step:
       │     ├─ tool step  → call MCP tool (loopback) → routed provider → result
       │     └─ chat step  → router picks cheapest model meeting threshold
       ├─ binds {task}, {step1.results}, … into subsequent step inputs
       └─ prints final result + cost + recipe path

Same recipe runs whether invoked via the CLI (frugal run) or via an MCP client calling a tool like frugal__chat with the matching use-case hint. The router, eval scoring, and pricing data are shared between both surfaces.

Two surfaces, one router

The CLI is for scripts, humans, CI, Makefiles, and ad-hoc tasks. The MCP server is for agents that already speak the Model Context Protocol — Claude Code, Claude Desktop, Cursor, custom hosts. Same routing engine underneath; agents call it via MCP, humans and scripts call it via CLI.

Three quality tiers — pick per invocation

Pass --quality cost for a quick extraction. --quality high for your agent's planner step. Default balanced for everything else.

high

Top-tier steps only. Planners, complex reasoning, novel code.

frugal run --quality high

balanced default

Best cost-quality tradeoff. The right choice ~80% of the time.

frugal run --quality balanced

cost

Cheapest viable recipe path. Classification, extraction, summaries.

frugal run --quality cost

Recipes that ship today

Each recipe is a deterministic step list in config/use_cases/<id>.yaml with three quality tiers and a labeled benchmark workload. The classifier picks one per invocation; the executor runs it; the result includes the recipe ID and the actual cost paid so you can audit the routing decision.

fresh-facts

Time-sensitive questions: today's news, current prices, this week's scores. Routes frugal__search + small chat synthesis.

research-synthesis

Long-form multi-source research. Chat today; rerank lands Phase 2.

code-dev

Code generation, debugging, review. Hosted chat now; local-model fallback in Phase 3.

factual-qa

Short factual lookups, trivia. Cheapest mini-tier model on the benchmark wins.

structured-extraction

Free text → JSON. Smallest JSON-mode-reliable model that passes the schema.

Install

One binary, ~10 MB. Frugal has no SaaS and issues no keys — it uses your existing OPENAI_API_KEY, ANTHROPIC_API_KEY, GOOGLE_API_KEY, TAVILY_API_KEY, SERPER_API_KEY from your environment. Adds frugal to your PATH. Release artifacts are signed with cosign; the installer verifies the checksum before moving the binary into place.

$ curl -fsSL https://frugal.sh/install | bash

From source

$ git clone https://github.com/brainsparker/frugal.git && cd frugal && make build

Install as an MCP server in your agent stack

$ frugal mcp install
# auto-detects Claude Desktop, Cursor, Claude Code and merges the
# 'frugal' MCP server into each detected config. Idempotent.

Migrating from v0.x? The OpenAI-compatible HTTP proxy and command-wrap mode were removed in v1.0 — see the v1.0 breaking-change note for the migration path. Pin FRUGAL_VERSION=v0.2.x in the installer to stay on v0.x if you need the legacy proxy.

Built to self-host

Your keys stay local
Provider API keys never leave your machine. Frugal reads them from the environment and forwards requests upstream; there is no control plane. Enterprise customers extend the same model to the analytics tier — self-hosted dashboard, zero data retention by architecture.
Signed releases
Every binary is cosign-signed (keyless, GitHub OIDC). The installer verifies the checksum and, when cosign is present, the signature before executing anything.
Observable by default
Structured logs, every recipe run prints its routing decision — recipe ID, step list, model picks, and the actual cost paid. frugal route previews the decision without executing.
Source-available, BUSL 1.1
Self-hosted and internal commercial use is permitted. Each version converts to Apache 2.0 four years after release.
