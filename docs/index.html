An AI toolchain router for your agents.

Frugal routes each task through the cheapest reliable path — search, chat, browser, code exec, or cache — exposed to your agent stack as frugal__* tools over MCP (the Model Context Protocol). One signed Go binary. Your keys. No account.

How Frugal decides

The expensive part of AI isn't the model. It's the wrong path — and the right path changes every request. Frugal makes the decision for you, with the actual cost in every response.

Classify
Detect the task and the capabilities required — local model, hosted model, search, browser, code execution, extraction, embeddings, semantic cache, or a multi-step combination.
Route
Pick the cheapest path that clears every quality threshold. One routing engine, shared by the CLI and the MCP server, so agents and scripts get the same answer.
Audit
Every response prints the decision — recipe, step list, model picks, actual cost. frugal route previews without paying.

One run, end to end

frugal run "summarize this repo"
       │
       ├─ classify task → recipe ID + quality tier
       ├─ load recipe steps from config/use_cases/<id>.yaml
       ├─ for each step:
       │     ├─ tool  → call MCP tool (loopback) → routed provider
       │     └─ chat  → router picks cheapest model that clears the threshold
       ├─ bind {task}, {step1.results}, … into subsequent step inputs
       └─ print result, cost, and recipe path

Same engine whether you type frugal run or your agent calls frugal__search via MCP. One routing decision, two surfaces.

MCP to ship it. CLI to try it.

Agent stacks — Claude Code, Cursor, Claude Desktop, custom hosts — talk to Frugal via MCP (Model Context Protocol). That's where Frugal lives in production. The CLI is for evaluation, scripting, and the install flow itself. Same routing engine underneath.

Three tiers. Pick per call.

--quality cost for extractions and lookups. --quality high for planners and novel code. Default balanced for everything else.

high

Top-tier steps only. Planners, complex reasoning, novel code.

frugal run --quality high

balanced default

Best cost-quality tradeoff. The right choice ~80% of the time.

frugal run --quality balanced

cost

Cheapest viable recipe path. Classification, extraction, summaries.

frugal run --quality cost

Toolchain components

Hosted chat from OpenAI, Anthropic, Google. Routed search via Tavily and Serper. Pricing synced from models.dev at startup; extend by editing ~/.frugal/config/models.yaml. Each component below ships only when the eval has data saying a toolchain built around it clears the quality bar.

Shipping — routes live traffic. Stubbed — schema slot exists; no executor wired. Planned — on the roadmap.

Hosted chat models shipping (internal)

OpenAI, Anthropic, Google chat completions inside recipe chat steps. Exposed as frugal__chat MCP tool in Phase 2.

Search API shipping

Routed cheapest web search per use case (Tavily, Serper) via the frugal__search MCP tool. Set TAVILY_API_KEY and/or SERPER_API_KEY.

Local-server-backed chat for the cheap path on summarize, code, extract (Phase 3).

Browser / fetch planned

Embeddings & vector search planned

Retrieval over user-supplied corpora to displace long-context.

Hash + similarity cache for repeated / near-duplicate questions.

Multi-step agent planned

Cheapest plan-and-call loop when one tool alone isn't enough.

Install

One Go binary, ~10 MB. No SaaS. No account. BYOK — Frugal reads OPENAI_API_KEY, ANTHROPIC_API_KEY, GOOGLE_API_KEY, TAVILY_API_KEY, SERPER_API_KEY from your environment and forwards calls upstream. Releases are cosign-signed; the installer verifies the checksum before moving the binary into place. 

$ curl -fsSL https://frugal.sh/install | bash

From source

$ git clone https://github.com/brainsparker/frugal.git && cd frugal && make build

Add Frugal to your agent

$ frugal mcp install
# auto-detects Claude Desktop, Cursor, Claude Code.
# merges 'frugal' into each detected config. idempotent.

Migrating from v0.x? The OpenAI-compatible HTTP proxy and command-wrap mode were removed in v1.0 — see the migration note. Pin FRUGAL_VERSION=v0.2.x to stay on the legacy proxy.

Built to self-host

Your keys stay local
Provider API keys never leave your machine. No control plane. Enterprise customers extend the same model to the analytics tier — self-hosted dashboard, zero data retention by architecture.
Signed releases
Every binary is cosign-signed (keyless, GitHub OIDC). The installer verifies the checksum and, when cosign is present, the signature.
Auditable routing
Every response prints the decision: recipe, step list, model picks, actual cost paid. frugal route previews without paying. Structured logs throughout.
