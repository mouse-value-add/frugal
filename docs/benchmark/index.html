An open benchmark for agent tool selection & routing.

AI apps don't fail because models are bad. They fail because they choose the wrong tools. The frugal benchmark scores how well an agent picks what to call — search or no search, which model tier, which toolchain — across cost, latency, and answer quality. The same engine that scores routes lives inside the frugal proxy, so what wins on the bench is what serves your next live request.

Decision quality, not just model quality

Most teams optimise model quality. Almost no one optimises decision quality. The result: expensive models get called for trivia, search gets fired on prompts the model could answer cold, and 60–80% of spend buys capability that wasn't needed. The benchmark asks a different question per prompt — was this the right tool, the right model, the right escalation? — and reports right vs wrong as a percentage.

Tool selection
For each prompt the workload labels whether search should fire. The benchmark scores did the agent invoke the right tool? and reports the share of correct calls.
Model routing
Three tiers compete: a near-free fallback, a Claude-class model, and a GPT-class frontier. The router tries cheap first and only escalates when the workload's confidence threshold isn't cleared.
Same engine, live
The scorer that decides "right tool, right tier" on the bench is the routing engine inside frugal serve. Bench wins ship.

Routing logic — cheap first, escalate only when required

Three tiers, simple rule: try the cheapest tier that has a fighting chance, escalate only when confidence drops below the workload threshold. The per-tier cards below quote curated rationale from config/use_cases/factual-qa.yaml — the bundle the proxy actually ships for short factual lookups today.

Tier 1 — try first

Cheap fallback · Gemini 2.0 Flash

Near-free per token. Adequate for factual recall, simple classification, JSON extraction.

Flash is the absolute floor — low per-token cost, adequate on factual recall.

Tier 2 — escalate

Claude-class · Claude Sonnet / Haiku

Strong reasoning at moderate cost. Wins on long-form synthesis and code edits where Tier 1 breaks down.

Sonnet is ~80% of Opus quality at 40% of the cost on long-form research tasks. Default choice for that use case.

Tier 3 — last resort

GPT-class frontier · GPT-4o / 4.1

Top-of-class reasoning. Reserved for prompts where Tier 2 still can't clear the workload's quality bar.

Even at the "high" tier most use cases don't need frontier models — only escalate when the bench says we have to.

What gets scored today

Cost is the ranking signal. Tool-selection correctness is the second signal. Latency and answer quality are captured today and will move into the ranking as the Pareto frontier becomes the deciding factor — they're roadmap, not vapor.

  • 1 · Cost Real per-token billing pulled from each provider's usage block. Aggregated as total run cost and savings vs the baseline tier.
  • 2 · Tool-selection correctness Workload labels each prompt with the expected tool decision (required, forbidden, optional). Reported as % of legs that matched the label.
  • 3 · Latency · captured, not yet ranking p50 / p95 wall-clock per request, plus optional time-to-first-token via --stream. Will join the Pareto frontier next.
  • 4 · Answer quality · captured, not yet ranking Deterministic scorers run today; an LLM-as-judge pass is opt-in via --judge-model and tracks hallucination rate separately.

Install

End users don't need to think about any of this. Install once and the proxy applies the same routing logic the benchmark scores — cheap first, escalate only when required — to every request. 

$ curl -fsSL https://frugal.sh/install | bash

Reproduce the bench

$ export OPENAI_API_KEY=...
$ frugal bench --out BENCHMARKS.md

Use-case-first run, with optional judge & TTFT

$ frugal bench \
    --use-case factual-qa \
    --quality balanced \
    --judge-model claude-opus-4-7-20250918 \
    --stream \
    --out BENCHMARKS.md

Workloads live at config/workloads/; bundles at config/use_cases/. Full report shape is documented inside BENCHMARKS.md.
