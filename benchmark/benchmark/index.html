What this is

The frugal benchmark is a lightweight evaluation system for agent decision-making.

  • Baseline: Always use the best model + search
  • Frugal: Use the cheapest viable option, escalate only when needed

Question answered: Are you overpaying for your AI stack?

Why this matters

  • Most teams optimize model quality, not decision quality
  • Overuse of expensive models and unnecessary tool calls
  • Higher latency and less predictable production behavior

What gets measured

1) Answer Quality

  • LLM-as-judge scoring
  • Task completion accuracy
  • Basic hallucination detection

2) Cost

  • Per-request cost
  • Total run cost
  • Cost vs baseline comparison

3) Latency

  • Time to first token
  • Total response time
  • Impact of tool calls

4) Tool Use Accuracy

  • Used search when needed?
  • Avoided tools when unnecessary?
  • Selected the right model?

  • Search was overused in simple reasoning tasks
  • Routing reduced cost with minimal quality loss

Who this is for

  • Teams building AI products with APIs
  • Engineers building agents and tool use flows
  • PMs balancing cost and UX

Run it yourself

The benchmark is part of the frugal CLI. It calls real provider APIs and writes a markdown report you can paste into a PR or commit at the root of your repo. You only need a key for whichever provider the workload's models use.

export OPENAI_API_KEY=...
go run ./cmd/frugal bench --out BENCHMARKS.md

To exercise all four dimensions — including LLM-judge scoring and time-to-first-token — point at one of the use-case bundles and add the optional flags:

go run ./cmd/frugal bench \
  --use-case factual-qa \
  --quality balanced \
  --judge-model claude-opus-4-7-20250918 \
  --stream \
  --out BENCHMARKS.md

Workloads live at config/workloads/; bundles at config/use_cases/. The full report shape is documented inside BENCHMARKS.md.
